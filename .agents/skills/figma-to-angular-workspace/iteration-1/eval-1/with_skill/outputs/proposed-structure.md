# Proposed feature structure (provisional)

This is the structural sketch the skill calls for in workflow step 3 (Plan the Angular structure). It is **provisional** until the Figma node `24:3285` is actually read — the concrete component names depend on what the frame contains. Two branches are listed (A and B); pick after extraction.

## Conventions reaffirmed (from the codebase + the skill)

- Folder template per feature: `src/app/features/<feature>/{data,feature,ui,util}`.
- Selector prefix: `app-` (from `angular.json`).
- Components: standalone, `ChangeDetectionStrategy.OnPush`, signal inputs/outputs, `inject()`, `@if`/`@for`/`@switch`.
- UI components (`ui/*`): presentational only. Signal `input()` / `output()`. No services, no router, no HTTP.
- Page components (`feature/*`): routable containers, hold state, inject services / signal stores.
- All SCSS uses `--mat-sys-*` tokens or `--app-*` gap-closed tokens. No raw hex outside `src/styles.scss` / `src/theme/_theme-colors.scss`.
- All new SCSS is mobile-first, with `@media (min-width: 768px)` and `@media (min-width: 1280px)` layers.
- Material primitives over custom HTML: `<mat-card>`, `<mat-flat-button>` / `<mat-stroked-button>` / `<mat-button>`, `<mat-fab>`, `<mat-chip>`, `<mat-icon>`, `<mat-form-field>`, `<mat-toolbar>`, `<mat-list>`, `<mat-tab-group>`, `MatDialog`, `MatSnackBar`.
- Customize Material via `mat.<component>-overrides()` in `src/styles.scss`; never `.mdc-*` selectors.

## Branch A — node 24:3285 turns out to be a posts feed (most likely)

The existing `src/app/features/posts/data/` already has a Post model, a Signal Store (`post-store.ts`), and an API service (`post-api.ts`). A feed-shaped frame slots into that data layer without new infrastructure.

```
src/app/features/posts/
├── data/                                       # exists — do not touch unless audit shows the model is missing fields
│   ├── infrastructure/post-api.ts
│   ├── models/post.model.ts                    # may need optional fields (cover, author, createdAt, …)
│   └── state/post-store.ts                     # NgRx Signal Store; reuse via inject(PostStore)
│
├── feature/
│   └── posts-page/                             # generated by: ng generate component features/posts/feature/posts-page
│       ├── posts-page.ts                       # container; injects PostStore; owns local UI signals (filter, search, selection)
│       ├── posts-page.html
│       ├── posts-page.scss                     # page-level layout, sticky header, FAB position, breakpoints
│       └── posts-page.spec.ts
│
├── ui/                                         # one folder per reused Figma component instance
│   ├── post-card/                              # if the frame shows a list of cards
│   │   ├── post-card.ts                        # input.required<Post>(), output<Post>() for tap/like/etc.
│   │   ├── post-card.html                      # <mat-card appearance="elevated">…</mat-card>
│   │   ├── post-card.scss
│   │   └── post-card.spec.ts
│   ├── posts-header/                           # only if the frame has a distinctive header beyond the global navbar/toolbar
│   ├── posts-empty-state/                      # only if state E5 says "build it" rather than "hide list"
│   └── …                                       # only add when the Figma frame shows reuse
│
└── util/                                       # only if there's date math, sorting, formatting — leave empty otherwise
```

### Routing (Branch A)

Append to `src/app/app.routes.ts`:

```ts
{
  path: 'posts',
  loadComponent: () =>
    import('./features/posts/feature/posts-page/posts-page').then(m => m.PostsPage),
},
```

Do **not** add a navbar entry without confirming with the developer (per `component-hierarchy.md`).

### State (Branch A)

- `PostsPage` injects `PostStore` via `inject(PostStore)`.
- Local UI-only signals (filter chip selection, search text, expanded item) live in the page component.
- No new signal store unless the frame introduces a state shape that doesn't fit `PostStore`.

## Generation commands (run only after audit + plan are signed off — NOT in this eval)

```bash
# Branch A — posts feed example
npx ng generate component features/posts/feature/posts-page --change-detection=OnPush --style=scss
npx ng generate component features/posts/ui/post-card        --change-detection=OnPush --style=scss
# Add more ui/* only for actual reused Figma component instances.

# Branch B — generic
npx ng generate component features/<feature>/feature/<feature>-page --change-detection=OnPush --style=scss
npx ng generate component features/<feature>/ui/<thing>             --change-detection=OnPush --style=scss
```

The Angular CLI generates `.ts`, `.html`, `.scss`, and `.spec.ts` per component — do not hand-create those files.

## What is NOT created up front

- No new NgModule (project is standalone-only).
- No new global SCSS file. New tokens go inside the existing `html { … }` block in `src/styles.scss`, annotated as `// gap-closed: …`.
- No `_theme-colors.scss` edits. That file is regenerated and hand-edits get overwritten.
- No `core/navbar` changes unless the developer explicitly asks for a navigation entry.
- No tests beyond the `.spec.ts` the CLI scaffolds; behavioral tests come after the user-story is locked.

## Validation gates (re-stated so they're not forgotten at the end)

1. `npx ng lint` — clean.
2. `npx ng build` — succeeds.
3. `playwright-cli` snapshot vs. Figma at 390×844, 768×1024, 1440×900.
4. Resize sweep 360 → 1920: no horizontal scroll, no layout breaks.
5. Dark mode snapshot at mobile + desktop.
6. Behavior check — exercise the user story end-to-end with the dev server running.
